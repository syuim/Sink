# 链接功能

通过仪表盘或已认证的 API 创建和管理链接。链接必须有目标 URL，其他所有行为均为可选。

## Slug 与标签

省略 Slug 时会生成一个小写的随机值。区分大小写模式只会改变**自定义** Slug 的规范化和匹配方式：它可以使 `Docs` 与 `docs` 成为两个不同的 Slug，但自动生成的随机 Slug 始终为小写。

标签会转换为小写并去重。每个链接最多支持 10 个标签，每个标签长度为 1–32 个字符。

## 过期时间与预览模式

创建或编辑链接时，如提供过期时间戳，该时间必须在未来。链接过期后将不再作为活动链接解析。导入则不同：为了不丢弃可移植的历史记录，它特意允许导入已经过期的记录。

预览模式是实例级演示设置。它将新建链接的有效期限制为五分钟，并阻止编辑或删除这些链接。请只在可随时弃用的公开演示中启用它。

## 密码与不安全警告

受密码保护的链接会向浏览器访客显示密码表单。API 风格的重定向客户端可以发送 `x-link-password`。通过当前创建和编辑流程设置的密码会先进行保护再存储，导出则会将已存储的密码值转换为可移植的受保护形式。KV 到 D1 数据迁移是例外：它会原样保留旧版密码值，因此较旧的迁移记录在密码被编辑前可能以明文形式存储在 D1 中。

设置链接的 `unsafe` 可以显式控制警告。如果配置了安全浏览，Sink 仅在**未提供** `unsafe` 时执行 DNS-over-HTTPS 检查。被拦截的 `0.0.0.0` 应答会将链接标记为不安全；查询失败时会放行。对于没有密码的不安全链接，必须通过带 `confirm=true` 的 `POST` 进行确认；`GET` Header 无法确认。若需以编程方式访问受密码保护的不安全链接，客户端可以同时发送 `x-link-password` 和 `x-link-confirm: true`。

## 目标路由

- **查询参数转发：** 在全局附加传入的查询参数，或按链接覆盖该设置。
- **地域路由：** 使用 Cloudflare 请求元数据，将两字母国家或地区代码映射到备用 URL。
- **设备路由：** 设置 Apple 或 Android 目标。匹配的设备目标优先于默认目标或地域目标。

## OpenGraph 与隐匿

自定义标题、描述和图片值会向识别到的社交爬虫提供链接预览。OpenGraph 图片可以上传到配置的 R2 存储桶。

隐匿功能会在全屏 iframe 中呈现 HTTPS 目标，同时让地址栏保留短 URL。它不是隐私功能：浏览器和开发者工具仍能看到目标。设置了严格 `X-Frame-Options` 或 `Content-Security-Policy: frame-ancestors` 的网站会拒绝加载，OAuth 或支付等敏感流程也可能拒绝被嵌入。

## 实例级重定向选项

Sink 可以更改普通重定向状态、附加 no-store Header、重定向首页以及重定向缺失的 Slug。这些设置影响整个实例；请在[配置参考](/zh-CN/configuration/#高级默认值)中查看其确切默认值和启用规则。
