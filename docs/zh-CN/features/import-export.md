# 导入/导出

Sink 的已认证 JSON 导入/导出 API 可在兼容实例之间传输链接记录。它们与 [KV 到 D1 数据迁移](/zh-CN/storage/kv-to-d1)和 [R2 链接快照](./backups)是不同操作。

## 导出

导出会以基于游标分页的 JSON 读取权威 D1 链接，包括过期记录。继续使用返回的游标，直到 `list_complete` 为 true。分页大小由[配置参考](/zh-CN/configuration/#构建-运行时公开覆盖值)中的批次设置控制。

可移植记录包括链接身份、时间戳、过期时间、路由、标签、OpenGraph 字段、安全和隐匿标志，以及受保护的密码表示形式。它不会暴露明文密码。

## 导入

每个导入请求最多接受已配置导出分页大小的一半。处理单个项目之前，会先对整个请求进行 Schema 验证；之后每个项目都会报告成功、跳过或失败。

重要行为：

- 允许并保留过期记录。
- 活动 Slug 冲突会被跳过，而非覆盖。
- Slug 会根据目标实例的自定义 Slug 大小写设置进行规范化。
- Sink 导出的可移植密码哈希可以在不知道明文密码的情况下导入。
- 仪表盘中经过遮盖的密码占位符不是有效的明文密码。
- 成功写入权威 D1 后，会尽力更新 KV 缓存。

导入不是完整的数据库恢复：它不会重建架构、墓碑记录、数据迁移运行状态或 KV 数据迁移标记。
