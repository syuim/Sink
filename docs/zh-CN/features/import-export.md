# 导入/导出

使用已登录的 JSON 导入/导出 API，在兼容的 Sink 实例之间迁移链接。这与[存储迁移](/zh-CN/storage/kv-to-d1)和 [R2 备份](./backups)不同。

## 导出

导出会分页下载数据库中的链接（每次响应是一页；用返回的游标继续请求，直到 `list_complete` 为 true）。每页大小由 `NUXT_PUBLIC_KV_BATCH_LIMIT` 控制。

每条记录包括短链码、URL、时间、路由、标签、社交预览字段、标志，以及**受保护**的密码形式 — 不含明文密码。

## 导入

每个请求最多接受导出分页大小的一半。Sink 先检查整份请求格式，再逐条报告成功 / 跳过 / 失败。

- 允许已过期记录
- 活动短链码冲突会跳过（不覆盖）
- 短链码遵循目标站点的大小写设置
- 从 Sink 导出的受保护密码可在不知道明文时导入
- 仪表盘里遮盖后的密码占位符**不是**有效密码

::: tip 不是完整恢复
导入不会重建整个数据库。不会重建删除标记、迁移历史或存储就绪标记。
:::
