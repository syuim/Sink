# 架构

Sink 在 Cloudflare Workers 或 Pages 上运行仪表盘、API 和短链接跳转。

## 有人打开短链接时发生什么

1. 访问者打开你域名下的短链接
2. Sink 查找链接并跳转（或显示密码/警告页）
3. 你在仪表盘或通过 API 管理链接（都需要登录）
4. 若启用了访问分析，访问会出现在报表和日志里

## Sink 用到的 Cloudflare 服务

| 绑定名称    | 产品             | 是否必需 | 白话说明                          |
| ----------- | ---------------- | -------- | --------------------------------- |
| `DB`        | D1               | 必需     | 主数据库 — 链接真正存放的地方     |
| `KV`        | KV               | 必需     | 加速跳转的缓存 + 一次性初始化标记 |
| `ANALYTICS` | Analytics Engine | 推荐     | 访问事件，供图表和日志使用        |
| `R2`        | R2               | 可选     | 文件存储，用于备份和社交图片      |
| `AI`        | Workers AI       | 可选     | 建议短链码和标题                  |

**D1** 是链接的真正存储。**KV** 是用于快速跳转的副本。保存链接后，Sink 会更新缓存；若缓存不对，会丢掉再从 D1 重新加载。

首次部署后，请打开一次 **Dashboard → Links**，完成存储初始化。在此之前，大多数链接 API 会提示「存储未就绪」（HTTP 423）。详见[存储初始化 / 迁移](/zh-CN/storage/kv-to-d1)。

R2 和 AI 是可选增强。请从[快速开始](./getting-started)开始。
