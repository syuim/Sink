# 架构

Sink 在 Cloudflare Workers 或 Pages 上运行仪表盘、API 和短链接重定向服务。

## 请求流程

1. 访问者打开 Sink 域名下的短链接。
2. Sink 读取链接，并返回配置的重定向或链接页面。
3. 仪表盘和外部客户端通过需要身份认证的 `/api/**` 路由管理链接。
4. 启用访问分析后，访问记录会用于仪表盘报表和日志。

## Cloudflare 服务

| 绑定        | 状态 | 用途                           |
| ----------- | ---- | ------------------------------ |
| `DB`（D1）  | 必需 | 存储链接及相关数据。           |
| `KV`        | 必需 | 加速链接重定向。               |
| `ANALYTICS` | 推荐 | 记录访问数据，用于分析和日志。 |
| `R2`        | 可选 | 存储备份和 OpenGraph 图片。    |
| `AI`        | 可选 | 生成建议的 Slug 和元数据。     |

D1 存储主要链接数据，KV 则用于快速查询重定向。通过仪表盘或 API 完成的链接变更会保存到 D1，并同步到 KV。

R2 和 Workers AI 是彼此独立的增强功能。即使不启用它们，Sink 仍可管理和重定向链接。部署要求请从[快速开始](./getting-started)阅读。
